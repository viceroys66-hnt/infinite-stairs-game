# 무한의 계단 (Quokka Stairs) — 제품 요구사항 문서

## 1. 개요

- **제품명**: 무한의 계단 (Quokka Stairs)
- **한 줄 설명**: 귀여운 쿼카가 무한히 계단을 오르는 아케이드 게임. 나무위키 기준 규칙(오르기/방향 전환 버튼, 체력 게이지)을 반영.

## 2. 목표 사용자

- 캐주얼 게임을 즐기는 사용자
- 짧은 시간에 즐길 수 있는 모바일/웹 게임 선호자

## 3. 핵심 기능

### 3.1 게임 플레이 (나무위키 규칙 반영)

- **캐릭터**: 귀여운 쿼카(Quokka). 캔버스로 그린 얼굴·몸·귀·미소.
- **조작**: 화면 하단 **두 개 버튼** — **방향 전환**(한 칸 올라가며 방향 바꿈), **오르기**(한 칸 올라감). 버튼을 누를 때마다 한 층씩 올라감.
- **계단**: 지그재그(좌·우 번갈아) 한 층당 한 발판. 올라갈수록 위에 새 계단이 보임.
- **체력 게이지**: 화면 상단 게이지가 시간에 따라 감소. **층이 올라갈수록 감소 속도가 빨라짐.** 게이지가 0이 되면 게임 오버.
- **점수**: 올라간 층 수(계단 수). 게임 오버 시 현재 층·최고 기록 표시.

### 3.2 게임 기록

- **최고 점수(High Score)** 저장 및 표시.
- 게임 시작 화면·게임 오버 화면에서 최고 점수 노출.
- 저장은 브라우저 로컬(동일 기기)에 유지.

### 3.3 화면 구성

- **시작 화면**: 타이틀, 쿼카, 안내 문구. **아래 두 버튼(방향 전환, 오르기) 중 하나를 누르면 시작.**
- **게임 화면**: 쿼카, 지그재그 계단, 상단에 **현재 층 수·체력 게이지·최고 기록**. 하단에 **방향 전환 / 오르기** 버튼.
- **게임 오버 화면**: 최종 층 수, 최고 기록. **다시 하기**는 두 버튼 중 하나를 누르면 재시작.

## 4. 비기능 요구사항

- 웹 브라우저에서 실행 (데스크톱·모바일).
- 별도 로그인 없이 로컬 저장만으로 동작.
- 문서화: PRD, spec, 구현 중 변경사항은 본 문서 또는 spec에 반영.

## 5. 제외 범위 (현재 버전)

- 온라인 순위표, 소셜 공유, 캐릭터/스킨 구매.
- 소리/배경음 (추후 추가 가능).

## 6. 성공 지표

- 한 번에 플레이 가능(시작 → 플레이 → 게임 오버 → 다시 하기).
- 최고 점수가 새로고침 후에도 유지됨.
- 규칙이 직관적으로 이해 가능.

## 7. 실행 및 테스트

- **실행**: `index.html`을 브라우저에서 **더블클릭해 바로 열어도 동작** (ES 모듈 없이 `game-bundle.js` 단일 스크립트 사용). 로컬 서버(`npx serve .` 등)도 가능.
- **테스트**: 프로젝트 루트에서 `node tests/run.js` 실행. (Node.js 필요, 기존 모듈 구조 테스트)
